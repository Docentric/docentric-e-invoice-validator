# Docentric e-Invoice Validator

[![Build Status](https://github.com/Docentric/docentric-e-invoice-validator/actions/workflows/build.yml/badge.svg)](https://github.com/Docentric/docentric-e-invoice-validator/actions/workflows/build.yml)
[![Docker Hub](https://img.shields.io/docker/v/docentric/e-invoice-validator?label=Docker%20Hub)](https://hub.docker.com/r/docentric/e-invoice-validator)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

A REST API and web-based validation tool for **ZuGFeRD** and **Factur-X** **e-invoices**, built with ASP.NET Core and React.

## Overview

This project provides a simple, user-friendly interface for validating ZuGFeRD and Factur-X **e-invoices** (both PDF and XML formats). It wraps the [Mustang Project](https://www.mustangproject.org/) validation engine in a modern web application with both API and UI access.

**Purpose**: This tool is primarily used for unit testing and quality assurance in the development of [Docentric AX](https://www.docentric.com/), our document generation and distribution solution. It helps ensure that e-invoices generated by Docentric AX comply with ZuGFeRD and Factur-X standards.

## Features

- **Web UI**: Modern React-based interface for uploading and validating invoices
- **REST API**: Programmatic access for automated testing and integration
- **Multiple Formats**: Supports both PDF (hybrid) and XML e-invoice formats
- **Standards Compliance**: Validates ZuGFeRD and Factur-X invoices using Mustang Project
- **Docker Ready**: Pre-built Docker images available on Docker Hub
- **CI/CD**: Automated builds and testing with GitHub Actions

## Technology Stack

- **Backend**: ASP.NET Core 10 (minimal API)
- **Frontend**: React + Vite + TypeScript
- **UI Components**: shadcn/ui with Tailwind CSS
- **Validation Engine**: Mustang Project CLI (Java-based)
- **Runtime**: .NET 10 + OpenJDK

## Quick Start

### Using Docker (Recommended)

The easiest way to run the validator is using Docker:

```bash
docker pull docentric/e-invoice-validator:latest
docker run -p 18889:8080 docentric/e-invoice-validator:latest
```

Then open your browser to `http://localhost:18889`

## Installation & Usage

### Prerequisites

For local development, you need:
- .NET 10 SDK
- Node.js 20.x or later
- Java Runtime Environment (JRE) 11 or later
- Git

### Building from Source

1. **Clone the repository**:
   ```bash
   git clone https://github.com/Docentric/docentric-e-invoice-validator.git
   cd docentric-e-invoice-validator
   ```

2. **Restore dependencies**:
   ```bash
   dotnet restore src/Docentric.EInvoice.Validator.slnx
   cd src/WebUI
   npm install
   cd ../..
   ```

3. **Build the solution**:
   ```bash
   dotnet build src/Docentric.EInvoice.Validator.slnx --configuration Release
   ```

4. **Run the application**:
   ```bash
   dotnet run --project src/Server/RestServer.csproj
   ```

The application will be available at `https://localhost:5001` or `http://localhost:5000`.

### Building Docker Image Locally

To build the Docker image yourself:

```powershell
# Windows
cd src
.\docker-build.ps1
```

```bash
# Linux/macOS
cd src
docker build -t docentric/e-invoice-validator:local .
```

Then run it:

```powershell
# Windows
.\docker-run.ps1
```

```bash
# Linux/macOS
docker run -p 8080:8080 docentric/e-invoice-validator:local
```

## API Usage

### Validate an e-Invoice

**Endpoint**: `POST /api/validate`

**Request**:
```bash
curl -X POST http://localhost:8080/api/validate \
  -F "file=@path/to/invoice.pdf"
```

**Response**:
```json
{
  "isValid": true,
  "validationMessages": [],
  "fileName": "invoice.pdf"
}
```

### Health Check

**Endpoint**: `GET /health`

```bash
curl http://localhost:8080/health
```

## Architecture

The solution consists of two main components:

### Backend (`src/Server`)
ASP.NET Core REST API that:
- Accepts file uploads (PDF or XML)
- Invokes Mustang Project CLI for validation
- Parses validation results
- Returns structured JSON responses

### Frontend (`src/WebUI`)
React SPA that:
- Provides a user-friendly upload interface
- Displays validation results
- Calls the REST API
- Built into static assets served from `Server/wwwroot` in production

### Docker Image
The Docker image bundles:
- .NET 10 runtime
- OpenJDK (for Mustang CLI)
- Mustang-CLI.jar in `/opt/Mustang-CLI.jar`
- Published ASP.NET Core application
- Built React SPA

## Development

### Running in Development Mode

1. **Start the backend**:
   ```bash
   cd src/Server
   dotnet run
   ```

   ```bash
   cd src/WebUI
   npm run dev
   ```

The frontend dev server runs on `http://localhost:5173` with hot reload enabled.

